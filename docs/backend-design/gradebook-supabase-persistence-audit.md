# Gradebook Supabase Persistence Audit

Updated: 2026-04-29

This inventory tracks the gradebook data that must survive sign-out/sign-in and cross-device use. The rule is: if it changes what a teacher sees in the gradebook, dashboard gradebook cards, assignment scoring, or assignment feedback, it must have a canonical Supabase write path and a canonical read/hydration path.

## Gaps Fixed In This Pass

- `saveScores()` now diffs against the persisted score snapshot and sends changed score rows to the canonical scoring RPCs. This covers UI paths that mutate the full score blob and only call `saveScores()`.
- `saveAssignmentStatuses()` now diffs against the persisted status snapshot and writes changed or removed status pills to `score.status`.
- `saveCategories()` now writes category adds, edits, deletes, and ordering to the `category` table, and patches assessment category IDs if Supabase returns a canonical UUID for a locally-created category.
- `saveLearningMap()` now writes subjects, competency groups, standards/sections, and tags to the normalized learning-map tables even when the calling UI does not call the newer `window.v2.*` RPC directly.
- When Supabase returns canonical IDs for learning-map entities, local references are patched so assignment `tagIds`, score `tagId`s, and student record keys do not keep stale local IDs.
- Rubric grading now writes the competency tag score that the UI actually reads, and mirrors to the rubric criterion row only when the clicked tag belongs to exactly one criterion.
- Rubric hydration now maps canonical `rubric_score.criterion_id` rows back to the criterion's linked competency tags before rebuilding the local score cache.
- Assignment collaboration metadata now persists from `saveAssessments()` for create/update flows, including mode changes, pair assignments, group assignments, and excluded students.

## Still Intentionally Local Or Derived

- Visual expansion/collapse state, temporary modal state, current filters, and transient undo stacks are UI state and should not be stored in Supabase.
- Locally generated score entry IDs are not canonical data. The canonical identity is `(enrollment_id, assessment_id, tag_id or criterion_id)`.
- Dashboard aggregate numbers are derived from canonical rows and should be recomputed from Supabase reads, not stored separately.
